# 프로젝트 최적화 리포트: PyTorch 기반 FCN-8s 의미 분할(Semantic Segmentation) 성능 개선 과정

본 리포트는 PASCAL VOC 데이터셋을 활용하여 VGG16 백본의 FCN-8s 모델을 학습시키는 과정에서 발생한 한계점들을 수학적·논리적으로 분석하고 단계적으로 해결해 나간 버전별 최적화 기록입니다.

## 버전별 성능 개선 요약

| 버전 | 핵심 변경 사항 | 결과 (mIoU) | 비고 |
| :--- | :--- | :--- | :--- |
| **v1.0** | 초기 아키텍처 설계 및 SGD 옵티마이저(Momentum 0.9) 적용 | 4.3% | 배경 편향(Local Minima) 문제 발생 |
| **v1.1** | Adam Optimizer 교체 (관성 파라미터 배제) | 18.5% | 배경 편향 탈출, 과적합 징후 확인 |
| **v1.2~1.3** | 좌우 반전 증강 및 StepLR(step_size=30) 스케줄러 최적화 | 20.52% | 학습률 소실 문제 해결 |
| **v2.0** | 글로벌 mIoU 계산법 도입 및 ImageNet 정규화 파이프라인 추가 | 24.51% | 평가 로직 오류 및 데이터 분포 불일치 교정 |
| **v3.0** | 마스크 완벽 동기화 Random Crop 데이터 증강 도입 | **32.27%** | 위치 불변성 학습 및 일반화 성능 극대화 |

## v1.0: 초기 모델 구축 및 '배경 편향' 현상 분석

* **구현 내용:** PyTorch 기반 VGG16 백본 FCN-8s 아키텍처 설계 및 SGD Optimizer(Momentum 0.9) 적용.
* **문제 발생:** 모델이 학습 오차를 줄이기 위해 이미지 내의 모든 픽셀을 '배경'으로만 예측하는 치명적인 편향 문제 발생.
* **수학적 원인:** PASCAL VOC 데이터셋의 극단적 클래스 불균형(배경 픽셀 80% 이상) 환경에서 SGD의 모멘텀이 악영향을 미침. "모든 픽셀을 배경으로 예측하라"는 지배적인 그래디언트가 관성으로 누적되면서, 모델을 늪과 같은 지역 최솟값(Local Minima)으로 더욱 빠르게 추락시킴.
* **SGD 업데이트 수식:** v_t = γ * v_{t-1} + η * g_t
$v_{t} = \text{이번에 이동할 최종 보폭과 방향(속도)}, \gamma = \text{모멘텀 계수}, v_{t-1} = \text{직전 스텝까지 누적해오던 속도와 방향}, \eta = \text{학습률}, g_{t} = \text{현재 기울기(현재 배치를 보고 계산해낸 기울기)}$

## v1.1: 적응형 학습률(Adaptive Learning Rate)을 통한 편향 탈출

* **수정 내용:** SGD를 폐기하고, 관성 파라미터인 1차 모멘트(β_1)를 0으로 배제하여 의도적으로 통제한 Adam Optimizer로 교체.
* **해결 및 결과:** 배경 편향을 성공적으로 탈출하고 비행기 등 객체의 형상 분할에 성공. (mIoU 4.3%에서 18.5%로 약 14.2% 대폭 상승)
* **수학적 원인:** 모멘텀 없이도 Adam이 성공한 핵심 이유는 기울기 제곱을 누적하는 2차 모멘트(v_t) 기반의 적응형 학습률 메커니즘. 압도적으로 빈번하게 등장하는 '배경' 파라미터는 업데이트 수식의 분모(√v_t)가 기하급수적으로 커져 스스로 학습 보폭에 브레이크를 검. 반면, 희귀한 객체 파라미터는 상대적으로 큰 보폭을 유지하여 데이터 불균형 패널티를 수학적으로 자체 상쇄함.
* **추가 문제:** 25 Epoch 이후 학습 Loss는 감소하나 검증 Loss가 상승하는 전형적인 과적합(Overfitting) 징후 확인.

## v1.2 ~ v1.3: 스케줄러 최적화를 통한 과적합 방어

* **문제 발생:** 과적합 방어를 위해 좌우 반전(H-flip) 데이터 증강 및 StepLR(step_size=5, gamma=0.1) 스케줄러를 도입했으나, 오히려 성능이 9.5%로 수직 하락.
* **디버깅 및 분석:** 학습 로그 분석 결과, 5 Epoch마다 학습률을 1/10로 줄이는 세팅이 너무 급격했음을 파악. 어려운 증강 데이터를 모델이 미처 학습하기도 전인 15 Epoch 부근에서 학습률이 거의 소실되어 최적화 불능 상태에 빠짐.
* **해결 및 결과:** StepLR(step_size=30)으로 학습률 감소 주기를 대폭 늘려 충분한 최적화 기간을 보장. (mIoU 9.5%에서 20.52%로 약 11.02% 성능 복구 및 상승)

## v2.0: 논리적 오류 교정 및 데이터 분포 정규화

* **수정 1 (평가 로직 정석 교정):** 배치 단위로 mIoU를 계산 후 평균을 내는 기존 방식이, 해당 배치에 등장하지 않은 클래스의 점수까지 0점 처리하여 전체 평균을 부당하게 깎아먹는 '평균의 함정' 오류 발견. 이를 혼동 행렬(Confusion Matrix)을 통해 전체 평가 과정의 픽셀 수를 누적한 뒤, 마지막에 단 한 번 IoU를 계산하는 '글로벌 mIoU 계산법'으로 교정함.
* **수정 2 (분포 불일치 해결):** ImageNet으로 사전 학습된 VGG16 가중치의 온전한 활성화를 위해, `dataset.py` 로더 단계에 ImageNet 공식 정규화(Mean/Std Normalize) 파이프라인 추가.
* **결과:** 올바른 채점 방식 도입과 데이터 분포 조정을 통해 그동안 억눌려있던 모델의 진짜 성능을 복구. (mIoU 20.52%에서 24.51%로 약 3.99% 상승)

## v3.0: 기하학적 데이터 증강을 통한 일반화 성능 극대화

* **목적:** PASCAL VOC 데이터셋의 절대적인 학습 데이터 부족(약 1,500장) 환경 극복.
* **구현 내용 (`dataset.py` 리팩토링):** 이미지를 320x320으로 1차 리사이징 후, 256x256 크기로 무작위 추출하는 Resize & Random Crop 기법 전면 도입. 이때 정답지에 해당하는 마스크(Label) 텐서와 원본 이미지의 Crop 좌표 및 H-flip 확률을 완벽하게 동기화 처리하는 파이프라인 구축.
* **기대 효과:** 1장의 원본 이미지에서 수천 가지의 무작위 구도를 생성해 냄으로써, 모델이 객체의 일부만 보고도 전체 문맥을 파악하는 '위치 불변성(Translation Invariance)' 학습 유도 및 과적합 원천 차단.
* **최종 결과:** mIoU 24.51%에서 32.27%로 약 7.76% 추가 상승. (초기 버전 v1.0 대비 총 약 28%의 성능 향상 달성)
